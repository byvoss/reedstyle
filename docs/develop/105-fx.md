# FX Namespace

Controls effects, animations, transforms, and transitions.

## Properties

### Transform

```html
<!-- Scale -->
<r-s as="div" fx="scale:1.1">        <!-- scale(1.1) -->
<r-s as="div" fx="scale:0.9">        <!-- scale(0.9) -->
<r-s as="div" fx="scale-x:1.5">      <!-- scaleX(1.5) -->
<r-s as="div" fx="scale-y:0.5">      <!-- scaleY(0.5) -->

<!-- Rotate -->
<r-s as="div" fx="rotate:45">        <!-- rotate(45deg) -->
<r-s as="div" fx="rotate:90">        <!-- rotate(90deg) -->
<r-s as="div" fx="rotate:180">       <!-- rotate(180deg) -->
<r-s as="div" fx="rotate:-45">       <!-- rotate(-45deg) -->

<!-- Translate -->
<r-s as="div" fx="translate-x:4">    <!-- translateX(1rem) -->
<r-s as="div" fx="translate-y:8">    <!-- translateY(2rem) -->
<r-s as="div" fx="translate-x:-4">   <!-- translateX(-1rem) -->

<!-- Skew -->
<r-s as="div" fx="skew-x:12">        <!-- skewX(12deg) -->
<r-s as="div" fx="skew-y:12">        <!-- skewY(12deg) -->

<!-- 3D Transforms -->
<r-s as="div" fx="rotate-x:45">      <!-- rotateX(45deg) -->
<r-s as="div" fx="rotate-y:45">      <!-- rotateY(45deg) -->
<r-s as="div" fx="perspective:1000"> <!-- perspective(1000px) -->

<!-- Combined transforms -->
<r-s as="div" fx="[scale:1.1, rotate:45, translate-x:4]">
```

## Performance Considerations

### GPU Acceleration

These properties use GPU acceleration:
- `transform` (scale, rotate, translate)
- `opacity`
- `filter`

Avoid animating:
- `width`, `height`
- `padding`, `margin`
- `top`, `left`, `right`, `bottom`

### Reduce Motion

ReedSTYLE respects `prefers-reduced-motion`:

```html
<!-- Animation disabled for users who prefer reduced motion -->
<r-s as="div" fx="animate:slide-up">
```

## Combining Effects

```html
<!-- Multiple effects -->
<r-s as="div" 
      fx="[scale:1.1, rotate:5, blur:sm, brightness:110]">

<!-- Hover + transition -->
<r-s as="div" 
      fx="[hover:scale:1.1, hover:rotate:5, transition:smooth]">

<!-- Animation + transform -->
<r-s as="div" 
      fx="[animate:fade-in, scale:0.9, transition:300ms]">
```

### Complex Effect Combinations

```html
<!-- Card with multiple effects -->
<r-s as="card" 
      fx="[shadow:light]"
      fx-hover="[hover:lift, shadow:normal, scale:small]"
      fx-scroll="[scroll:fade-in, delay:small]"
      fx-click="[click:ripple, color:primary]">
  <h3>Interactive Card</h3>
  <p>Multiple effects working together</p>
</r-s>

<!-- Hero Section with Parallax -->
<r-s as="hero" fx="[scroll:parallax, speed:slow]">
  <r-s as="content" fx="[scroll:fade-in, delay:normal]">
    <r-s as="h1" fx="[scroll:slide-up, delay:small]">
      Welcome
    </r-s>
    <r-s as="p" fx="[scroll:slide-up, delay:normal]">
      Stunning visual effects
    </r-s>
    <r-s as="button" 
          fx="[hover:glow, intensity:bright]"
          fx-click="[click:pulse]">
      Get Started
    </r-s>
  </r-s>
</r-s>

<!-- Gallery with Staggered Animations -->
<r-s as="gallery" layout="[grid:3, gap:4]">
  <r-s as="card" 
        fx="[scroll:zoom-in, stagger:tiny]"
        fx-hover="[hover:lift, transition:smooth]">
    <img src="1.jpg" />
  </r-s>
  <r-s as="card" 
        fx="[scroll:zoom-in, stagger:tiny]"
        fx-hover="[hover:lift, transition:smooth]">
    <img src="2.jpg" />
  </r-s>
  <r-s as="card" 
        fx="[scroll:zoom-in, stagger:tiny]"
        fx-hover="[hover:lift, transition:smooth]">
    <img src="3.jpg" />
  </r-s>
</r-s>
```

## Best Practices

1. **Use semantic effect names** that describe the visual result
2. **GPU-accelerated properties** for performance (transform, opacity)
3. **Respect reduced motion** preferences automatically
4. **Subtle effects** are usually better than dramatic ones
5. **Test performance** on slower devices and mobile
6. **Combine effects thoughtfully** to avoid overwhelming users
7. **Use Visual Scope** for consistent intensity levels
8. **Progressive enhancement** - CSS effects work without JavaScript
9. **Lazy initialization** for scroll effects with Intersection Observer
10. **Cache effect calculations** for better performance
