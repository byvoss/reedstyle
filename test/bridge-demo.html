Bridge Layer Demo

This page demonstrates the ReedSTYLE Bridge Layer system for migrating from other CSS frameworks.

How it Works

Configuration

Enable frameworks in reedstyle.bridge.yaml:

bridge:
  bootstrap:
    enabled: true
    path: "./node_modules/bootstrap/dist/css/bootstrap.min.css"

CSS Layer Structure

The bridge layer sits between settings and theme:

@layer settings, bridge, theme, free;

This ensures ReedSTYLE styles win over framework styles.

Migration Strategy

  1. Enable your existing framework in bridge config
  2. Gradually replace framework classes with ReedSTYLE attributes
  3. Add overrides for specific conflicts
  4. Once migration complete, disable framework

Example: Bootstrap Migration

Before (Bootstrap)

<div class="card">
  <div class="card-body">
    <h5 class="card-title">Card</h5>
    <p class="card-text">Content</p>
    <a href="#" class="btn btn-primary">Button</a>
  </div>
</div>

After (ReedSTYLE)

<r-s as="card">
  <h3>Card</h3>
  <p>Content</p>
  <r-s as="button-primary">Button</r-s>
</r-s>

Supported Frameworks

  • Bootstrap 5.x
  • Tailwind CSS
  • Material UI
  • Ant Design
  • Bulma
  • Foundation
  • Custom/Legacy CSS
